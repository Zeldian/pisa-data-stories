# PISA Data Inventory

## Data access hierarchy

Future story agents should use the **smallest** dataset that supports the analysis:

```
1. School Summary  →  pisa_{cycle}_school_summary.csv
      One row per school. Pre-aggregated weighted means and SDs per domain.
      Use this for any analysis that works at the school level.

2. Core Extract    →  pisa_{cycle}_core.csv.gz
      One row per student. Identifiers, final weight, PV1-10 for all domains.
      Use when student-level variation is needed (distributions, subgroup analyses).

3. Raw Data        →  data/raw/SPSS_STU_QQQ.zip  (2018 primary)
      Full SPSS file with all variables including replicate weights.
      Use only when: (a) variables outside the core extract are required,
      or (b) BRR standard errors are needed (requires W_FSTURWT1-80).
```

Do not open raw files if the processed datasets cover your needs.

## Processed file reference

### Core extract — `pisa_{cycle}_core.csv.gz`

One row per student. Generated by `scripts/prepare_pisa_cycle.py`.

| Column | Description |
|--------|-------------|
| `CYCLE` | PISA cycle year (constant within file) |
| `CNT` | Country code (3-char ISO) |
| `CNTSCHID` | International school ID |
| `CNTSTUID` | International student ID |
| `W_FSTUWT` | Student final weight |
| `PV1MATH`–`PV10MATH` | Math plausible values |
| `PV1READ`–`PV10READ` | Reading plausible values |
| `PV1SCIE`–`PV10SCIE` | Science plausible values |

**Replicate weights** (`W_FSTURWT1`–`W_FSTURWT80`) are **not** included. Load from
raw SPSS if BRR standard errors are required.

```python
import pandas as pd
df = pd.read_csv("data/processed/pisa_2022_core.csv.gz")
```

### School summary — `pisa_{cycle}_school_summary.csv`

One row per school. Derived from the core extract by `scripts/prepare_pisa_cycle.py`.

| Column | Description |
|--------|-------------|
| `CYCLE` | PISA cycle year |
| `CNT` | Country code |
| `CNTSCHID` | International school ID |
| `N_STUDENTS_SAMPLED` | Raw count of students in the sample |
| `SUM_W_FSTUWT` | Sum of student weights (weighted population in school) |
| `MEAN_MATH` | Weighted mean math score |
| `SD_MATH` | Weighted population SD of math scores |
| `MEAN_READ` | Weighted mean reading score |
| `SD_READ` | Weighted population SD of reading scores |
| `MEAN_SCIE` | Weighted mean science score |
| `SD_SCIE` | Weighted population SD of science scores |

**Methodology note:** Domain scores per student are the unweighted mean of the 10
plausible values (a point estimate, not full Rubin's-rules pooling). School-level
means and SDs are then weighted by `W_FSTUWT`. This is appropriate for exploration,
initial hypothesis checks, and school-level summaries. For rigorous statistical
inference, use the core extract with proper PV pooling per `data/methodology.md`.

```python
import pandas as pd
sch = pd.read_csv("data/processed/pisa_2022_school_summary.csv")
```

## PISA 2018 — primary cycle

### Source

Downloaded from the OECD PISA 2018 database file server:
`https://webfs.oecd.org/pisa2018/`

Official database page (Cloudflare-protected, requires a browser):
`https://www.oecd.org/pisa/data/2018database/`

Downloaded: 2026-06-29

### Files in `data/raw/` (2018)

| File | Size | Description |
|------|------|-------------|
| `SPSS_STU_QQQ.zip` | 478 MB | Student questionnaire — all background variables + plausible values |
| `SPSS_SCH_QQQ.zip` | 3.0 MB | School questionnaire |
| `SPSS_TCH_QQQ.zip` | 13 MB | Teacher questionnaire |
| `PISA2018_codebook.xlsx` | 5.2 MB | Variable codebook |

**`SPSS_STU_QQQ.zip`** — primary file for all current story analyses.
~612,000 student records across 79 countries. Key variables:
- `PV1READ`–`PV10READ`, `PV1MATH`–`PV10MATH`, `PV1SCIE`–`PV10SCIE`
- `W_FSTUWT` + `W_FSTURWT1`–`W_FSTURWT80` (final weight + 80 BRR replicate weights)
- `CNT`, `CNTRYID`, `CNTSCHID`, `CNTSTUID`
- `ESCS`, `ST004D01T` (gender), `ST001D01T` (grade), derived indices

**2018 has no core extract.** Stories load the raw SPSS file directly and select
only the columns they need. This is intentional — the 2018 workflow was established
before the cross-cycle system and is not changed.

Additional 2018 files **not yet downloaded** (fetch only when required):
- `SPSS_STU_COG.zip` (~477 MB) — cognitive item-level responses

## PISA 2022, 2015, and 2012 — cross-cycle robustness

These cycles are used only for robustness checks on 2018 findings.
Do not replace 2018 as the primary dataset.

### Variable harmonization

Core identifier and weight variables are **identical** across 2015, 2018, and 2022.
PISA 2012 and earlier used different identifier names; the pipeline renames them:

| Variable in core extract | 2022/2018/2015 raw name | 2012 raw name |
|--------------------------|-------------------------|---------------|
| `CNTSCHID` | `CNTSCHID` | `SCHOOLID` |
| `CNTSTUID` | `CNTSTUID` | `STIDSTD` |
| `CNT` | `CNT` | `CNT` |
| `W_FSTUWT` | `W_FSTUWT` | `W_FSTUWT` |

**Plausible value count:** PISA 2015+ provides **10 PVs per domain** (PV1–PV10).
PISA 2012 provides **5 PVs per domain** (PV1–PV5). The core extract includes
whichever PVs are present; the `variables_included` field in the inventory records
the actual range.

**Country coverage differs across cycles.** Do not assume 2018 countries appear in all cycles.

## How to download / regenerate

### PISA 2022 (automated)

```bash
python3 scripts/prepare_pisa_cycle.py --cycle 2022
```

Downloads from `https://webfs.oecd.org/pisa2022/STU_QQQ_SPSS.zip` (~651 MB),
then generates the core extract and school summary automatically.

### PISA 2015 (manual browser download required)

The OECD 2015 data page is Cloudflare-gated and cannot be accessed programmatically.

1. Open in a browser: `https://www.oecd.org/pisa/data/2015database/`
2. Download the SPSS student questionnaire file (look for "Student questionnaire
   data file" in SPSS format — typically `PUF_SPSS_COMBINED_CMB_STU_QQQ.zip`, ~400–500 MB)
3. Run:
   ```bash
   python3 scripts/prepare_pisa_cycle.py --cycle 2015 --raw-zip ~/Downloads/<filename>.zip
   ```

### PISA 2012 (requires free OECD account registration)

OECD requires registration to download pre-2015 PISA files. The data is free but
not publicly accessible via direct URL — there is no bypass.

1. Register at: `https://www.oecd.org/en/data/register.html`
2. Download from: `https://www.oecd.org/pisa/data/2012database/`
   (look for the SPSS student questionnaire file)
3. Run:
   ```bash
   python3 scripts/prepare_pisa_cycle.py --cycle 2012 --raw-zip ~/Downloads/<filename>.zip
   ```

**Comparability note:** PISA 2012 has 5 plausible values per domain (vs. 10 for 2015+).
Analyses using PV pooling should account for this; the school summary uses a simple
mean and is not affected. Country coverage in 2012 was 65 economies (vs. ~79 in 2018).
The 2012 major domain was mathematics (same as 2022), making it suitable for
math-focused robustness checks alongside 2018 and 2022.

### Idempotency

The script skips any step whose output already exists. Use `--force` to regenerate:

```bash
python3 scripts/prepare_pisa_cycle.py --cycle 2022 --force
```

Check `data/processed/pisa_cycles_inventory.csv` before running — raw files are
hundreds of MB each.

## Methodology

See [`methodology.md`](methodology.md) for required analysis procedures: plausible
value pooling (Rubin's rules), student weights, BRR standard errors, and benchmarking.
These apply to every analysis, including cross-cycle comparisons.

The school summary files use a simplified PV approach (mean of 10 PVs per student,
not full pooling). This is sufficient for exploration but not for publishable estimates.
